================================================================================
                           TYPOGRAPHY HIERARCHY
================================================================================

H1 - 34px (extrabold, -0.5 letter-spacing)
     ╔═══════════════════════════════════════════╗
     ║ Welcome Back, User                        ║
     ║ Very Large Hero Title                     ║
     ╚═══════════════════════════════════════════╝

H2 - 32px (extrabold, -0.4 letter-spacing)
     ╔═══════════════════════════════════════════╗
     ║ Page Title                                ║
     ║ Main Content Header                       ║
     ╚═══════════════════════════════════════════╝

H3 - 26px (extrabold, -0.3 letter-spacing)
     ┌───────────────────────────────────────────┐
     │ Section Title                             │
     │ Category or Major Subheading              │
     └───────────────────────────────────────────┘

H4 - 22px (bold)
     ┌───────────────────────────────────────────┐
     │ Card Title or Subsection                  │
     └───────────────────────────────────────────┘

H5/H6 - 20px/18px (bold)
     Standard heading sizes for cards and lists

Body (14px, normal weight, relaxed line height)
     This is body text. It should be comfortable to read
     with plenty of breathing room. Line height is relaxed
     at 1.5x for natural, easy reading.

BodyLg (15px) - Larger body text
BodySm (13px) - Smaller descriptions

Label (12px, bold, uppercase, +0.8 letter-spacing)
     FIELD LABEL OR CAPTION

Hint/Muted (13px, gray color)
     Small helper text or secondary information

Error (13px, red color)
     Error message shown to user

Success (13px, green color)
     Success confirmation message

Accent (varies, accent color)
     Highlighted or interactive text

================================================================================
                        SHADOW/ELEVATION SYSTEM
================================================================================

shadowSmall (elevation: 2)
  offset: 0, 1    opacity: 0.08    blur: 3px
  Use: subtle card, hover states

shadowMedium (elevation: 4)
  offset: 0, 2    opacity: 0.10    blur: 6px
  Use: cards, modals, buttons

shadowLarge (elevation: 8)
  offset: 0, 4    opacity: 0.15    blur: 12px
  Use: important modals, featured sections

shadowXl (elevation: 12)
  offset: 0, 8    opacity: 0.20    blur: 16px
  Use: fullscreen modals, overlays

Stacking (visual hierarchy):
  Base (no shadow)  ↓
  shadowSmall       ↓ (cards)
  shadowMedium      ↓ (modal)
  shadowLarge       ↓ (important)
  shadowXl          ↓ (fullscreen)

================================================================================
                              KEY TAKEAWAYS
================================================================================

✓ Use 8pt multiples for all spacing (4, 8, 16, 24, 32, 48)
✓ Use TEXT_STYLES presets instead of inline font sizes
✓ Use ICONS constants instead of hardcoded emojis
✓ Use LAYOUT_PRESETS for common patterns
✓ Maintain clear h1-h6 hierarchy
✓ Keep spacing consistent within sections
✓ Use semantic icons from organized categories
✓ Apply appropriate shadows for depth
✓ Never use magic numbers - use constants

CONSISTENCY = BEAUTY = PROFESSIONALISM

================================================================================
                          QUICK DECISION GUIDE
================================================================================

Question: What spacing should I use?
Answer:   Use SPACING constants: xs(4), sm(8), md(16), lg(24), xl(32), xxl(48)

Question: What size should this text be?
Answer:   Use TEXT_STYLES: h1-h6 for headings, body/bodySm for text, label for captions

Question: Which icon should I use?
Answer:   Find in ICONS config: music, action, status, ui, greeting categories

Question: What should this layout look like?
Answer:   Use LAYOUT_PRESETS: centerFull, rowBetween, rowCenter, card, section

Question: Should I add custom styling?
Answer:   Only if it's not covered by design system. Prefer presets!

Question: How do I make things consistent?
Answer:   Use design system constants. Never hardcode values!
